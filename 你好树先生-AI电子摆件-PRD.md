## 你好树先生 — AI 电子摆件 产品需求文档

**版本：** v1.1  
**日期：** 2026-04-02  
**实现方式：** HTML / CSS / JavaScript 单页应用  

---

### 一、产品概述

"你好树先生"是一款像素风 AI 电子摆件，核心画面是一棵随四季变换外观的梧桐树。它不只是装饰——树会感知真实天气并做出视觉响应，每隔一段时间会有应季小动物来到树旁，以对话气泡的方式讨论当下的热点新闻。动物离开后，梧桐树会短暂地浮现自己的"思考与总结"，让这个摆件既有美感，又有信息密度。

整体定位：放在桌面或小屏幕上的"有生命的信息艺术品"。

---

### 二、核心体验目标

1. **有生命感** — 树、天气、动物构成一个微型生态，时刻在变化。
2. **有信息感** — 小动物的对话承载真实热点，用户瞥一眼就能知道世界在发生什么。
3. **不打扰** — 所有信息以自然节奏流过，不弹窗、不提醒，像窗外的风景。

---

### 三、视觉风格

整体采用像素艺术（Pixel Art）风格，画面分辨率建议以 320×240 或 480×320 为基准进行像素绘制，再整数倍放大到实际显示尺寸。调色板以低饱和、自然色系为主，四季各有一套主色调：

| 季节 | 主色调 | 氛围关键词 |
|------|--------|------------|
| 春 | 嫩绿、樱粉、淡黄 | 生机、温柔 |
| 夏 | 深绿、湛蓝、金黄 | 繁茂、热烈 |
| 秋 | 橙红、金黄、棕褐 | 温暖、沉静 |
| 冬 | 银白、灰蓝、深棕 | 清冷、宁静 |

---

### 四、功能模块详述

#### 4.1 梧桐树主体

梧桐树是整个画面的视觉锚点，常驻画面正中偏下位置。树的外观随季节变化：

- **春**：枝条抽出嫩芽，逐渐展开小叶，偶有花瓣飘落的粒子效果。
- **夏**：树冠丰满浓绿，叶片层叠，伴有光斑闪烁动画。
- **秋**：叶片渐变为橙红金黄，持续有落叶飘下的动画。
- **冬**：枝干裸露，偶有残叶，枝头可能积雪。

季节的判断逻辑：每天零点（或页面加载时）通过 `Date` API 获取当前日期，按以下规则映射季节——3-5月为春，6-8月为夏，9-11月为秋，12-2月为冬。季节切换时树的外观以渐变动画过渡，过渡时长建议 5-10 秒。

#### 4.2 天气系统

每小时通过大模型联网搜索获取用户当前所在城市的实时天气（用户在配置中填写城市名称）。大模型返回结构化的天气状态关键词，前端据此映射为以下视觉效果，叠加在整个场景之上：

| 天气状态 | 视觉效果 |
|----------|----------|
| 晴天 | 背景偏暖亮色，阳光粒子从右上角斜射，树冠有光斑 |
| 多云 | 像素云朵缓慢飘过，背景微微灰暗 |
| 阴天 | 整体色调压低，背景灰调 |
| 小雨 | 雨滴粒子从上方飘落，地面出现涟漪动画 |
| 大雨/暴雨 | 雨滴加密，雨势加大，树枝微微摇晃 |
| 雪 | 雪花粒子缓慢飘落，地面逐渐积雪 |
| 风 | 树枝、树叶摇摆动画加强，落叶/雪花/雨滴倾斜飘动 |
| 雾/霾 | 画面叠加一层半透明灰白遮罩 |

天气效果可以叠加（比如"大风+雨"），树的动画应与天气联动（雨天叶片下垂，风天枝条摆动）。

#### 4.3 小动物与新闻对话

这是产品的信息核心。每小时出现 3 次小动物，即每隔 20 分钟来一次，每次停留约 10 分钟后离开。

**动物与季节对应：**

| 季节 | 动物 | 像素形象简述 |
|------|------|-------------|
| 春 | 小鸟 | 圆润的像素鸟，停在枝头或树下，说话时翅膀微扇 |
| 夏 | 昆虫（如瓢虫/蝴蝶） | 在树叶间或树干上活动，说话时触角抖动 |
| 秋 | 松鼠 | 抱着坚果坐在树根旁，说话时尾巴摆动 |
| 冬 | 雪兔 | 白色小兔蹲在雪地上，说话时耳朵抖动 |

**对话机制：**

每次对话由两个 AI 角色完成，它们共享同一段新闻上下文，但承担不同职能：

- **角色 A — 新闻播报者（动物）：** 负责抓取最新热点新闻，以简短口语化的方式播报给"树"听。每次播报 1-2 条新闻，每条凝缩为 1-2 句话（约 15-30 字）。
- **角色 B — 评论者（树）：** 负责对新闻做出短评、提问或感慨，语气带有"老树"的从容与智慧感。每次回复 1-2 句话（约 15-30 字）。

一次完整对话流程如下：

```
[00:00] 动物入场动画（从画面边缘走入/飞入/跳入）
[00:05] 动物打招呼（气泡："嘿，听说了吗？"）
[00:15] 动物播报第1条新闻（气泡）
[00:30] 树回应评论（气泡，显示在树冠附近）
[00:50] 动物播报第2条新闻（气泡）
[01:10] 树回应评论（气泡）
[01:30] 动物或树发出闲聊/感慨（气泡）
...（可根据对话内容继续 2-4 轮）
[09:00] 动物告别（气泡："下次再聊！"）
[09:30] 动物离场动画
[10:00] 对话结束
```

对话以气泡形式展示，气泡为像素风格圆角矩形，动物的气泡在画面左下/右下（随动物位置），树的气泡在树冠旁。气泡显示 8-12 秒后淡出。对话轮次之间间隔 15-30 秒，营造自然交谈的节奏。

**对话频率设定：** 每次停留 10 分钟内，预计交互 6-10 轮气泡（含打招呼和告别），平均每 60-90 秒一个气泡，节奏不紧不慢。

#### 4.4 树的思考与总结

每次动物对话结束并离场后，树进入"沉思"状态，持续约 1 分钟：

- 视觉表现：树冠微微发光（或叶片闪烁柔光），营造"思考中"的氛围。
- 出现一段总结文字，以特殊样式的气泡（比如虚线边框或淡金色底）显示在树冠上方。
- 总结内容由大模型生成，输入为本次对话涉及的新闻内容，输出为 1-2 句富有哲思或温暖的总结（约 20-40 字）。例如："世界总在变，树根处却一直安静。大事小事，且听风说。"
- 总结气泡显示 40-50 秒后淡出，之后场景恢复平静。

#### 4.5 背景与昼夜系统

根据用户当前时间自动调整画面明暗：

- **日出/日落：** 背景天空由暖橙渐变，6:00-7:30 日出，17:30-19:00 日落（可按实际经纬度计算精确时间）。
- **白天：** 明亮天空，浅蓝或有云。
- **夜晚：** 深蓝/深紫色天空，点缀像素星星，月亮按月相变化。树的颜色整体压暗。
- **深夜（23:00-6:00）：** 画面最暗，动物仍按计划来访，但对话频率可降低（可选）。

#### 5.2 外部 API 依赖

本产品仅依赖一个外部服务——支持联网搜索的大模型 API。天气查询和新闻获取均通过大模型的 web search 能力完成，无需单独接入天气或新闻 API，大幅降低了接入复杂度和多 API Key 管理成本。

| 服务 | 推荐方案 | 调用频率 | 用途 |
|------|----------|----------|------|
| 大模型（联网搜索） | OpenAI API（GPT-4o-mini w/ web search） / 其他支持联网搜索的模型 API | 见下方明细 | 天气查询、新闻抓取、对话生成、总结生成 |

**每小时大模型调用明细：**

| 调用类型 | 频率 | 说明 |
|----------|------|------|
| 天气搜索 | 1 次/小时 | 搜索当前城市实时天气，返回结构化天气关键词 |
| 新闻搜索 + 动物对话 | 3 次/小时 | 每 20 分钟 1 次，搜索热点新闻并生成动物播报对话 |
| 树的评论回复 | 3 次/小时 | 每轮对话中树的回应（可与新闻搜索合并为一次多轮调用） |
| 树的沉思总结 | 3 次/小时 | 每轮对话结束后生成总结 |
| **合计** | **约 7-10 次/小时** | 通过合并多轮对话可优化至 7 次/小时 |

#### 5.3 大模型调用设计

所有外部数据获取和内容生成均通过同一个大模型 API 完成。模型需支持联网搜索（web search / tool use）能力，以便实时获取天气和新闻信息。

**Prompt 设计要点：**

**天气查询 Prompt（每小时 1 次）：**
```
请搜索{城市名称}当前的实时天气情况。
只需要返回一个 JSON，格式如下，不要返回其他内容：
{
  "weather": "晴/多云/阴/小雨/大雨/雪/雾",
  "wind": "无/微风/大风",
  "temp": 25
}
```

**新闻搜索 + 动物播报 Prompt（每 20 分钟 1 次）：**
```
你是一只{当前季节对应动物}，住在一棵老梧桐树旁边。你性格活泼好奇。
请先搜索今天最新的 2 条热点新闻，然后以口语化、简短的方式播报给
梧桐树听（每条不超过 30 字），像在和老朋友聊天。
同时为每条新闻准备一句打招呼的话和一句告别的话。
请以 JSON 格式返回：
{
  "greeting": "打招呼",
  "news": [
    {"headline": "新闻原标题", "report": "口语化播报"},
    {"headline": "新闻原标题", "report": "口语化播报"}
  ],
  "farewell": "告别语"
}
```

角色 B（评论者/树）的 System Prompt 示例：
```
你是一棵有着百年历史的老梧桐树，性格沉稳温和、富有哲思。
小动物会来给你讲新闻，你需要做出简短的回应（不超过30字），
可以是感慨、提问或温和的评论。语气从容，偶有幽默。
```

树的总结 Prompt 示例：
```
你是一棵沉思中的老梧桐树。刚才听了以下新闻对话，请用一句富有
哲理或温暖的话总结你的感受（20-40字），风格像是老树的内心独白。
```

每次调用使用较低的 `max_tokens`（约 80-200）以控制输出长度和成本。`temperature` 建议设为 0.8，保持一定创造性。天气查询的 `temperature` 设为 0，确保返回准确的结构化数据。

#### 5.4 定时调度逻辑

```
主循环 (requestAnimationFrame)
├── 每帧：渲染动画（树摇摆、粒子、天气效果）
├── 每秒：检查对话气泡显示/淡出
├── 每分钟：检查昼夜状态更新
├── 每20分钟：触发小动物入场 → LLM搜索新闻 → 生成对话 → 生成总结
├── 每小时：LLM搜索天气 → 更新天气视觉效果
└── 每天0点：检查日期更新季节
```

### 六、对话时序详细设计

以一个小时为单位的完整时间线：

```
:00  ─── 动物 #1 入场 ───
:00 ~ :10  对话进行中（6-10轮气泡交互）
:10  ─── 动物 #1 离场 ───
:10 ~ :11  树的沉思与总结（1分钟）
:11 ~ :20  安静时段（只有树和天气动画）

:20  ─── 动物 #2 入场 ───
:20 ~ :30  对话进行中
:30  ─── 动物 #2 离场 ───
:30 ~ :31  树的沉思与总结
:31 ~ :40  安静时段

:40  ─── 动物 #3 入场 ───
:40 ~ :50  对话进行中
:50  ─── 动物 #3 离场 ───
:50 ~ :51  树的沉思与总结
:51 ~ :00  安静时段（等待下一轮）
```

每次动物来访抓取不同的新闻，避免重复。同一小时内的 3 次来访可以是同一种动物的不同个体，也可以视觉上做微小的差异化（如颜色深浅略有不同）。

### 七、像素资源清单

需要设计和制作的像素美术资源：

**梧桐树（4 季 × 每季 2-3 帧动画）：** 春树（发芽态、嫩叶态）、夏树（茂密态）、秋树（变色态、落叶态）、冬树（裸枝态、积雪态）。

**小动物（4 种 × 每种 4 组动画）：** 入场动画（3-5帧）、待机/说话动画（2-3帧循环）、倾听动画（2帧）、离场动画（3-5帧）。

**天气粒子：** 雨滴、雪花、落叶、花瓣、光斑——每种 2-3 帧。

**背景天空：** 晴天、多云、阴天、夜晚——各一张背景图，支持平滑过渡。

**UI 元素：** 像素风对话气泡（普通款、树的思考款）、字体推荐使用像素字体（如 "Zpix" 中文像素字体）。

### 九、MVP 阶段与迭代计划

**Phase 1 — 静态场景（MVP）：** 实现梧桐树的四季静态像素图渲染，根据当前日期自动切换季节，实现基本的昼夜背景切换。预计工时 2-3 天。

**Phase 2 — 天气联动：** 接入大模型搜索天气，将返回的天气关键词映射为雨、雪、风、晴等粒子效果和背景变化，树的动画与天气联动。预计工时 2-3 天。

**Phase 3 — 动物与对话：** 实现小动物的出入场动画，通过大模型搜索新闻并生成角色对话，完成对话气泡系统和树的总结功能。预计工时 4-5 天。

**Phase 4 — 打磨与部署：** 优化动画流畅度，调整对话节奏，增加像素美术细节，处理 API 错误兜底，适配不同屏幕尺寸。预计工时 2-3 天。

### 十、风险与兜底

**API 调用失败：** 大模型 API 不可用时，天气保持上一次缓存的结果（默认晴天）；新闻对话从预置的趣味对话库中随机抽取（如动物和树聊季节变化、自然趣事等非时效性话题），确保画面始终有内容流动。

**成本控制：** 全部依赖单一大模型 API，每小时调用约 7-10 次，每次 token 消耗较少（天气查询约 100 tokens，新闻对话约 300-500 tokens，总结约 200 tokens）。按 GPT-4o-mini 定价估算，日均成本低于 $1。如使用其他更经济的模型，成本可进一步降低。

**隐私：** 用户仅需配置城市名称（非精确坐标），不涉及地理定位权限。所有 API 调用均在用户浏览器端直接发起。

---

### 十一、成功指标

作为个人电子摆件，成功的衡量更偏向体验感受：画面在桌面上放一天是否依然让人觉得舒服而不烦；瞥一眼能否自然地获取到当下的新闻信息；树和动物的对话是否偶尔能让人会心一笑。技术层面，动画帧率应稳定在 30fps 以上，API 调用不出现明显延迟或报错。
